## 🛠️ Tech Stack & Architecture Decisions

The system was designed with a serverless architecture on Google Cloud Platform (GCP) to ensure horizontal scalability and cost-efficiency.

* **Languages:** Python 3.10+ (Polling engine and ETL), TypeScript (API logic in Cloud Functions).
* **Protocols:** SNMP v2c/v3 (Extraction of specific OIDs from Zebra/Ricoh hardware).
* **Database:** **Google Cloud Firestore**. We chose NoSQL (Document-based) due to the semi-structured nature of printer error logs and the need for horizontal scalability. Firestore allows us to store failure history as subcollections for each asset, enabling fast time-series queries.

## 🔥 Data Engineering Challenges Overcome

The logistics environment introduced critical network reliability and performance challenges, which were mitigated through pure backend logic:

### 1. Debounce Logic & Data Sanitization
Printers operating in a logistics environment generate momentary network instabilities (false logs). We implemented a **state buffer** in the Python backend. The system only registers a true status change (e.g., Online -> Error) if the condition persists for 'n' polling cycles.
* **Result:** Without this, the database would be polluted with hundreds of false logs, invalidating cost and uptime reports.

### 2. Hardware Rate Limiting & Concurrency Control
Thermal printers (Zebra ZT411) have limited processing power and can freeze if they receive too many simultaneous SNMP requests. We implemented concurrency control and request scheduling in the Python collection agent.
* **Result:** We ensured that the monitoring system wouldn't accidentally trigger a Denial of Service (DoS) attack on the very assets we were protecting.

## 📊 Core Features & Business Intelligence

The system transforms raw telemetry into actionable visualizations for Selbetti/Mercado Livre management:

1. **Digital Twin (Heatmap):** Real-time geographical and connectivity status visualization for all operational assets.
2. **Predictive Maintenance:** Logic that calculates thermal head wear based on accumulated mileage telemetry, anticipating failures before they halt operations.
3. **Cost Auditing:** Consolidated report that crosses real printed volume with estimated supply costs, enabling precise financial management.
